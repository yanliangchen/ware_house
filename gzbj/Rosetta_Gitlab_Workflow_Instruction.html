Local Configuration

  • Git global user name and email setting
$ git config --global user.name "<Your Name>"
$ git config --global user.email "<email@example.com>"
  • To connect to the remote server, you can use ssh or https.
  1. If you prefer to use ssh, you need ssh key and to store your public key in the remote git repository.

To generate ssh key

$ ssh-keygen -t rsa -C "<email@example.com>"

You can check existing ssh key with:

$ ls -al ~/.ssh

Once the key is generated, go to your git server SSH Keys section and add the public SSH key to the server. Then you will be able to connect to the server.

  1. If you would like to use https, please use the https://... address of repository as the remote url. and you may want to store your credential on your OS, please follow Git Tools - Credential Storage.
  • Unix/Linux use 0x0A (LF) as newline character, DOS/Windows use 0x0D0A (CRLF) as newline character. We should make sure source code files using LF when they are pushed to GitLab.
# Convert to LF when submit, don't convert when checkout
$ git config --global core.autocrlf input
# Reject to submit file which contains both LF and CRLF
$ git config --global core.safecrlf true

Code Management with Git

To see current branches in git,

$ git branch

You have only one branch at the start, this is master branch. To do your work create your own branch in your local machine,

$ git branch <yourname>

Change (Checkout) to your own local branch and do all your changes only there,

$ git checkout <yourname>

Do your changes only in yourname branch and commit as you want,

$ git add --all
